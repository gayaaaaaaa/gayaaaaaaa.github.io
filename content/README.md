# 內容管理說明

這個資料夾用於管理部落格的所有內容，使用 YAML 格式儲存。

## 資料夾結構

- `notes/` - 筆記文章，每個 YAML 檔案對應一篇筆記

## 筆記格式

在 `notes/` 資料夾中建立 YAML 檔案，例如 `數學課-三角函數.yaml`：

```yaml
id: "1"
title: "筆記標題"
categoryId: "1"  # 對應到類別 ID
date: "2024-01-15T10:00:00Z"
excerpt: "筆記摘要"
content: |
  # 筆記標題
  
  筆記內容可以使用 **Markdown** 語法撰寫。
  
  - 支援列表
  - 支援**粗體**和*斜體*
  - 支援程式碼區塊
  
  ```javascript
  console.log("Hello, World!");
  ```
```

**重要**：
- **檔案名稱就是 URL slug**：檔案名稱（不含 `.yaml` 或 `.yml` 副檔名）會自動用作筆記 URL（例如：`數學課-三角函數.yaml` → `/notes/數學課-三角函數`）
- 檔案名稱必須是唯一的，建議使用有意義的中文名稱
- `categoryId` 必須對應到現有的類別 ID（1-5）
- `content` 欄位支援完整的 Markdown 語法
- `date` 使用 ISO 8601 格式（例如：`2024-01-15T10:00:00Z`）

## 類別

目前支援的類別：
- `1`: 數學
- `2`: 英文
- `3`: 歷史
- `4`: 科學
- `5`: 地理

## 生成內容

每次修改 YAML 檔案後，執行以下命令重新生成內容：

```bash
npm run content:generate
```

或者在開發時，內容會自動生成：

```bash
npm run dev
```

或者在建置時，內容會自動生成：

```bash
npm run build
```

## Markdown 支援

所有 `content` 欄位都支援完整的 Markdown 語法：

- 標題（# ## ###）
- **粗體** 和 *斜體*
- 列表（有序和無序）
- 程式碼區塊和行內程式碼
- 引用
- 連結和圖片
- 表格（透過 GFM 擴展）

## 注意事項

1. **檔案名稱就是 URL**：檔案名稱（不含 `.yaml` 或 `.yml` 副檔名）會自動成為筆記的 URL slug
2. 檔案名稱必須是唯一的，建議使用有意義的中文名稱
3. 每次修改內容後記得執行 `npm run content:generate` 或重新啟動開發伺服器
4. **所有編輯和新增功能已移除**：筆記只能透過編輯 YAML 檔案來管理
